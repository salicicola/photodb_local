Salicicola Core (Names Package)

It includes SQLite tables defined in names/models.py: names_name, names_speciesmeta, names_nameannotation, and names_commonname. The latter serves as index and maybe removed in future versions (or left empty). The possibility to report bugs disabled in this configuration.

Editing records available by standard Django Admin interface or special form to edit two linked tables simultaneously.

The obligatory modules include files: views.py, checklist.py, and invasive.py, together with referenced files in "templates/names" folder, as well as supporting files in /static/* or data/static/* folders.

They will allow to view updated version of The Vascular Plants of Massachusetts. A County Checklist (simplified, without recently added county records), together and its subsets of invasive plants. If running as standalone application, all links may still work but lead to salicicola server (i.e., access to Internet is needed).

Using most of these features, needs a login with proper permissions. If running this app first time, a temporary user name 'admin' with a password 'admin' may be supplied; otherwise, run a command in application ROOT directory: python manage.py createsuperuser and follow instructions (replace python to python3 if necessary).
